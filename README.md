# Lean Six Sigma DMAIC Case Study

## Server Diagnostic Process Optimization

> **Professional Portfolio Project | Lean Six Sigma Green Belt | Process Improvement | Data Analysis**

---

## Project Overview

This project demonstrates the application of the **Lean Six Sigma DMAIC (Define, Measure, Analyze, Improve, Control)** methodology to optimize a simulated server diagnostic process in a data center repair environment.

The objective was to improve process efficiency by reducing diagnostic turnaround time, minimizing process variation, increasing first-pass success, and reducing unnecessary retests through a standardized diagnostic workflow.

This project was completed as part of a **Lean Six Sigma Green Belt** portfolio and showcases the use of statistical analysis, process improvement methodologies, and quality engineering tools to solve an operational problem using a structured, data-driven approach.

---

# Business Problem

Server diagnostic operations often experience delays caused by inconsistent troubleshooting procedures, repeated diagnostic cycles, and variation in technician decision-making.

These issues can lead to:

- Increased diagnostic turnaround time
- Higher process variation
- Additional retests
- Reduced operational efficiency

This case study demonstrates how Lean Six Sigma tools can be applied to identify these issues, validate improvements using statistical analysis, and establish controls to sustain long-term performance.

---

# Project Objectives

The project focused on three primary objectives:

- Reduce average diagnostic turnaround time
- Improve first-pass success rate
- Reduce the average number of retests per server

Success was evaluated using measurable operational KPIs before and after the proposed process improvement.

---

# DMAIC Summary

| Phase | Activities |
|--------|------------|
| **Define** | Project Charter, Team Charter, Problem Statement, Project Scope |
| **Measure** | Data Collection Plan, Process Mapping, KPI Definition |
| **Analyze** | Descriptive Statistics, Hypothesis Testing, Pareto Analysis, Histogram, Boxplot |
| **Improve** | Standardized Diagnostic Workflow, Troubleshooting Standardization |
| **Control** | KPI Monitoring, Process Audits, Corrective Action Plan |

---

# Project Results

| Performance Measure | Before | After | Improvement |
|---------------------|--------:|-------:|------------:|
| Average Diagnostic Turnaround Time | 47.03 min | 35.57 min | **24.4% Reduction** |
| Process Variation (Standard Deviation) | 4.35 | 2.99 | **31.3% Reduction** |
| First-Pass Success Rate | 73.3% | 90.0% | **+16.7 Percentage Points** |
| Average Retests per Server | 1.60 | 1.10 | **31.3% Reduction** |

# Process Overview

The process begins when a failed server is received for diagnosis. Technicians review diagnostic information, identify the failure category, perform repairs, and execute validation testing. If validation fails, the server undergoes additional troubleshooting before being retested.

The project identified this rework loop as a major contributor to increased turnaround time.

# Root Cause Analysis

A Fishbone (Cause-and-Effect) Diagram was developed to identify the primary contributors to extended diagnostic turnaround time.

Major contributing factors included:

- Inconsistent troubleshooting procedures
- Lack of standardized work instructions
- Process variation between technicians
- Repeated diagnostic cycles
- Limited process monitoring
